# 文章模块后端开发文档

## 架构概述

文章模块采用 **Human-in-the-Loop** 设计 + **5 个智能体编排**，AI 在关键节点暂停等待用户确认后再继续。

### 阶段状态机

```
PENDING → TITLE_GENERATING → TITLE_SELECTING → OUTLINE_GENERATING → OUTLINE_EDITING → CONTENT_GENERATING → COMPLETED
                                  ↑ 用户确认标题                          ↑ 用户确认大纲
```

- `status` 字段追踪粗粒度生命周期（`PENDING → PROCESSING → COMPLETED/FAILED`）
- `phase` 字段追踪细粒度阶段（`ArticlePhaseEnum`），带 `can_transition_to()` 校验
- `TITLE_SELECTING` 和 `OUTLINE_EDITING` 为两个用户介入点

### 三阶段异步执行

| 阶段 | 触发 API | 执行内容 | SSE 结束事件 |
|------|---------|---------|-------------|
| 阶段1 | `POST /article/create` | Agent 1 生成 3-5 组标题方案 | `TITLES_GENERATED` |
| 阶段2 | `POST /article/confirm-title` | Agent 2 流式生成大纲 | `OUTLINE_GENERATED` |
| 阶段3 | `POST /article/confirm-outline` | Agent 3+4+5 正文、配图、合成 | `ALL_COMPLETE` |

每个阶段通过 `asyncio.create_task` 异步执行，独立的 SSE 连接推送进度。

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/article/create` | 创建文章任务（触发阶段1：标题生成） |
| POST | `/api/article/confirm-title` | 确认标题 + 补充描述（触发阶段2） |
| POST | `/api/article/confirm-outline` | 确认大纲（触发阶段3） |
| POST | `/api/article/ai-modify-outline` | AI 根据建议修改大纲 |
| GET | `/api/article/progress/{task_id}` | SSE 进度推送 |
| GET | `/api/article/{task_id}` | 获取文章详情 |
| POST | `/api/article/list` | 分页查询文章列表 |
| POST | `/api/article/delete` | 删除文章（软删除） |

全部接口需要登录态。

### SSE 消息类型

| 类型 | 说明 |
|------|------|
| `TITLES_GENERATED` | 标题方案生成完成，携带 `titleOptions`（等待用户选择） |
| `AGENT2_STREAMING` | 大纲流式输出 |
| `OUTLINE_GENERATED` | 大纲生成完成，携带 `outline`（等待用户编辑） |
| `AGENT3_STREAMING` | 正文流式输出 |
| `AGENT3_COMPLETE` | 正文生成完成 |
| `AGENT4_COMPLETE` | 配图需求分析完成 |
| `IMAGE_COMPLETE` | 单张配图完成 |
| `AGENT5_COMPLETE` | 配图生成完成 |
| `MERGE_COMPLETE` | 图文合成完成 |
| `ALL_COMPLETE` | 全部完成 |
| `ERROR` | 错误 |

## 数据库表

SQL 建表语句见 `sql/create_table.sql`。

```sql
CREATE TABLE article (
    id                   bigint auto_increment PRIMARY KEY,
    taskId               varchar(64) NOT NULL UNIQUE,
    userId               bigint NOT NULL,
    topic                varchar(500) NOT NULL,
    userDescription      text,
    enabledImageMethods  json,
    style                varchar(20),
    mainTitle            varchar(200),
    subTitle             varchar(300),
    titleOptions         json,
    outline              json,
    content              text,
    fullContent          text,
    coverImage           varchar(512),
    images               json,
    status               varchar(20) DEFAULT 'PENDING',
    phase                varchar(50) DEFAULT 'PENDING',
    errorMessage         text,
    createTime           datetime DEFAULT CURRENT_TIMESTAMP,
    completedTime        datetime,
    updateTime           datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    isDelete             tinyint DEFAULT 0
);
```

状态流转（粗粒度）：`PENDING → PROCESSING → COMPLETED / FAILED`
阶段流转（细粒度）：`PENDING → TITLE_GENERATING → TITLE_SELECTING → OUTLINE_GENERATING → OUTLINE_EDITING → CONTENT_GENERATING`

## 开发期间发现并修复的问题

### 1. article_router 未注册
- `routers/__init__.py` 未导出 `article_router`
- `main.py` 未 `include_router`
- 已在两处补充注册

### 2. ImageMethodEnum 缺少枚举值
- 原只有 `PEXELS`、`PICSUM`
- 补充了 `MERMAID`、`ICONIFY`、`EMOJI_PACK`、`NANO_BANANA`、`SVG_DIAGRAM`

### 3. article 表未建
- 通过 `sql/create_table.sql` 创建

### 4. agent1 标题解析错误
- Prompt 返回 3-5 个标题的数组，但代码当单个对象解包
- 修改为遍历数组，生成 `List[TitleOption]` 存入 `state.title_options`，首项作为 `state.title`

### 5. agent4 JSON 解析层级错误
- LLM 返回 `{contentWithPlaceholders, imageRequirements}` 整体对象
- 代码直接遍历了外层对象而非 `imageRequirements` 数组
- 改为先提取 `response_data["imageRequirements"]` 再遍历

### 6. `_parse_json_response` 不能处理 markdown 代码块
- LLM 返回的 JSON 被 ` ```json ``` ` 包裹
- 添加正则提取代码块内容，并处理 `None` 输入

### 7. `_call_llm` 可能返回 None
- `message.content` 可能为 `None`
- 改为 `return response.choices[0].message.content or ""`

### 8. 缺失方法补齐
- `_insert_image_after_section` — 图文合成中匹配章节插入配图
- `_build_streaming_data`、`_build_image_complete_data`、`_build_complete_message_data` — SSE 消息构建

## AI 配置

使用阿里云百炼 DashScope API，OpenAI 兼容模式：

```python
client = AsyncOpenAI(
    api_key=settings.dashscope_api_key,
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)
```
