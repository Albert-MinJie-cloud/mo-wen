## API 接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/health` | 健康检查 | 无 |
| POST | `/api/user/register` | 用户注册 | 无 |
| POST | `/api/user/login` | 用户登录 | 无 |
| POST | `/api/user/logout` | 用户登出 | 可选登录 |
| GET | `/api/user/get/login` | 获取当前登录用户 | 登录 |
| GET | `/api/user/get?id=` | 根据 ID 获取用户 | 无 |
| POST | `/api/user/list/page` | 分页查询用户列表 | 管理员 |
| POST | `/api/user/add` | 添加用户 | 管理员 |
| POST | `/api/user/update` | 更新用户 | 管理员 |
| POST | `/api/user/delete` | 删除用户（逻辑删除） | 管理员 |

## 核心流程

### 鉴权机制

```
请求 → Cookie(SESSION) → get_session_id() → get_current_user() → require_login() / require_admin()
                                                 ↓
                                         Redis 读取 Session
```

- `get_session_id`：从 Cookie 提取 session_id（可选）
- `get_current_user`：根据 session_id 从 Redis 获取用户信息（可选）
- `require_login`：要求必须登录，否则抛出 `NOT_LOGIN_ERROR`
- `require_admin`：要求必须是管理员，否则抛出 `NO_AUTH_ERROR`

### 登录流程

1. 用户提交账号密码 → `UserService.login()`
2. 验证密码 → 生成 `session_id`（UUID）
3. 用户信息存入 Redis → 设置 Cookie `SESSION=session_id`
4. 返回 `LoginUserVO`

### 登出流程

1. 从 Cookie 获取 `session_id`
2. 调用 `remove_session()` 清除 Redis 中的 Session 数据
3. 删除浏览器 Cookie
