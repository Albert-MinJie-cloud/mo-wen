# 人机交互增强

在AI应用开发中，有一种设计理念叫做Human in the Loop（让人类参与到AI决策中）

## 任务状态机流程图

```mermaid
flowchart LR

    OUTLINE_GENERATING -->|大纲生成完成| OUTLINE_EDITING

```

## 状态流转明细表

```text
状态枚举名	节点类型	中文说明	进入触发条件	流转下一状态
PENDING	初始状态	等待处理	创建新任务	TITLE_GENERATING
TITLE_SELECTING	用户介入	等待选择标题	标题方案生成完成	OUTLINE_GENERATING（用户确认标题后）
OUTLINE_GENERATING	AI 执行	生成大纲中	用户确认选定标题	OUTLINE_EDITING
OUTLINE_EDITING	用户介入	等待编辑大纲	大纲生成完成	CONTENT_GENERATING（用户确认大纲后）
CONTENT_GENERATING	AI 执行	生成正文中	用户确认大纲内容	COMPLETED
COMPLETED	终态	任务完成	正文与配图全部生成完毕	无
```

## 线性流程梳理（快速通读）

创建任务 → 自动进入「待处理」初始状态
AI 自动生成多组标题方案 → 进入「待用户选择标题」状态
用户确认最终标题 → AI 自动生成完整文章大纲 → 进入「待用户编辑大纲」状态
用户确认 / 修改大纲后提交 → AI 自动生成正文内容与配图
全部内容生成完毕 → 任务进入完成终态

## 接口设计

| 接口                            | 说明       | 触发条件            |
| ------------------------------- | ---------- | ------------------- |
| POST /article/confirm-title     | 确认标题   | TITLE_SELECTING阶段 |
| POST /article/confirm-outline   | 确认大纲   | OUTLINE_EDITING     |
| POST /article/ai-modify-outline | ai修改大纲 | OUTLINE_EDITING     |

## sse 消息扩展

| 消息类型          | 说明                               |
| ----------------- | ---------------------------------- |
| TITLES_GENERATED  | 标题方案生成完成，携带titleOptions |
| OUTLINE_GENERATED | 大纲生成完成，携带outline          |
